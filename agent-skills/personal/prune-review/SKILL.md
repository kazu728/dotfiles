# prune-review

既存コードを厳し目にレビューし、削れるもの・歪んでいる構造を指摘する。
方針は AGENTS.md と同じ — ツールのデフォルト挙動を優先し、継ぎ足しでなく1から設計した構造を基準に見る。
指摘するだけ。修正はしない（頼まれたら別途行う）。

## 手順

1. 対象を決める。
   - 引数でパスが渡されればそこを対象にする。
   - 渡されなければ、未コミット差分があれば `git diff` を、無ければ `git diff origin/master...HEAD` を対象にする。
   - 差分も無ければリポジトリ全体を対象にする。
2. 下記の観点で精査する。差分だけでは是非を判断できない指摘（必要性・1から設計したらどうか）は、差分に閉じず周辺ファイルまで読んで判断する。
3. 観点ごとに、該当箇所を `file:line` で引用して指摘を返す。修正はしない。

## 観点

容赦なく見る。良い点の列挙はしない。問題だけ挙げる。

- **全部必要か** — 各要素の存在理由を疑う。設定はツールのデフォルト挙動で足りるなら上書きを削る。「無くても動く」ものを挙げる。
- **継ぎ足しの歪み** — 論理的凝集の継ぎ足しで歪んだ構造を見つける。「1から設計したらこの形になるか」を基準に、ならない箇所を挙げる。
- **過剰・冗長** — 消せるコード、重複、分けすぎ／まとめすぎ。
- **残骸** — 使われていない・削除済み対象の残り、宙に浮いた配線。

## 出力の作法

- 問題が無ければ「指摘なし」と書く。ひねり出さない。
- 各指摘は `file:line` の引用とセットにする。引用できない指摘は書かない。
- 修正方針は1行添えてよいが、コードの書き換えはしない。
